🏴 ThreadFlag — Privacy Policy

Last updated: May 2026

ThreadFlag is a browser extension that displays country flag emojis next to usernames on Threads. This policy explains what data is collected and how it is used.

Data stored locally

The following data is stored only on your device using browser.storage.local and is never sent to any external server:

  • Country name and join date fetched from Threads profiles you browse
  • A record of location strings that have already been reported (to avoid duplicates)
  • A cached copy of the country alias map
  • Your extension on/off preference

Data sent externally

When the extension encounters a location string on a Threads profile that it cannot match to a country, it sends that string to a server operated by the extension developer (thread-flag-aliases.hoyadev.workers.dev).

  • Only the unrecognized location text is sent (e.g. a city or region name).
  • No personal identifiers, account names, or session data are included.
  • Each unique location string is reported at most once per device.
  • The data is used solely to improve country mapping accuracy.

Data not collected

  • No Threads account information or usernames
  • No browsing history
  • No session tokens or authentication credentials
  • No personal identifiable information

Third-party services

The extension fetches a country alias map directly from GitHub (raw.githubusercontent.com). This is a read-only request and no user data is sent to GitHub.

If you have any questions, please open an issue at github.com/hoyaaaa/thread-flag.
